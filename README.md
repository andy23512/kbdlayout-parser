# kbdlayout-parser

Parse data from [kbdlayout.info](https://kbdlayout.info/) into deterministic artifacts used by this repository.

## What This Repository Generates

This project currently maintains two data pipelines.

### 1) Language Tree Pipeline

Input:

- `res/languages.html`

Outputs:

- `res/languages-tree-data.json`
- `res/languages-tree-data.ts`

These artifacts contain a tree of languages/groups and their layout references.

### 2) Keyboard Layout Pipeline

Inputs:

- `res/index.html`
- `res/drivers/*.html`
- `res/driver-xmls/*.xml`

Outputs:

- `res/keyboard-layouts.ts`

Notes:

- `res/keyboard-layouts.ts` is generated by `yarn convert-layouts`.
- `res/keyboard-layouts.json` is a checked-in reference artifact and is not currently regenerated by any script.

## Requirements

- Node.js 18+ (for built-in `fetch`)
- Yarn 4 (project uses `yarn@4.9.2`)

Install dependencies:

```bash
yarn install
```

## Scripts

### Fetch Source Pages

- `yarn fetch-index`
  - Download homepage HTML to `res/index.html`.
- `yarn fetch-languages`
  - Download language tree page HTML to `res/languages.html`.
- `yarn fetch-driver-pages`
  - Parse `res/index.html` for driver links and download each page to `res/drivers/*.html`.
- `yarn fetch-driver-xmls`
  - Parse `res/index.html` for driver links and download each XML to `res/driver-xmls/*.xml`.

### Generate Language Tree Artifacts

- `yarn parse-languages-tree`
  - Parse `res/languages.html` and regenerate both language-tree artifacts.
- `yarn test-languages-tree`
  - Validate parser determinism and error-handling behavior.
- `yarn verify-languages-tree`
  - Verify contract invariants and equality between JSON and TS artifacts.

### Generate Keyboard Layout Artifacts

- `yarn convert-layouts`
  - Parse downloaded driver pages/XML files and regenerate `res/keyboard-layouts.ts`.

## Typical Workflows

### Refresh Language Tree Data

```bash
yarn fetch-languages
yarn parse-languages-tree
yarn test-languages-tree
yarn verify-languages-tree
```

### Refresh Keyboard Layout Data

```bash
yarn fetch-index
yarn fetch-driver-pages
yarn fetch-driver-xmls
yarn convert-layouts
```

## Troubleshooting

- `yarn fetch-driver-pages` fails with missing `res/index.html`
  - Run `yarn fetch-index` first. `fetch-driver-pages` reads driver links from `res/index.html`.

## Language Tree Data Contract

`res/languages-tree-data.json` root is an array of nodes.

Node shape:

- `kind`: `"group"` or `"language"`
- `code`: language/group code
- `name`: display name
- `layouts`: layout references
- `children`: present only when `kind === "group"`

Layout shape:

- `id`: required deterministic ID (generated by shared `src/layout-id.ts` logic)
- `name`: layout display name
- `isDefault`: default layout marker
- `isDefaultFromOtherLanguage`: inherited-default marker

## Breaking Changes (Recent)

- Root wrapper was removed.
  - Old format: `{ "version": 1, "source": "res/languages.html", "nodes": [...] }`
  - New format: `[...]`
- Layout `id` is now required.
  - ID generation is shared with keyboard layout conversion through `src/layout-id.ts`.
- Layout `klid` and `href` were removed from language tree layout objects.
  - Use `id` and `name` as canonical identifiers/labels in downstream consumers.
